Doctor
User Story 1

As a doctor, I want to access my patients’ past medical records so that I can diagnose and treat them accurately.
Priority: High
Story Points: 5

Acceptance Criteria:

Doctor can search and select a patient profile.

System displays previous treatments, prescriptions, and reports.

Information appears in a clear and organized format.

Only authorized doctors can view patient history after login.

User Story 2

As a doctor, I want to modify and update prescription details so that patients always receive correct and up-to-date medication instructions.
Priority: High
Story Points: 3

Acceptance Criteria:

Doctor can edit, add, or remove medications from a patient’s prescription.

The updated prescription becomes instantly available to the patient.

System maintains logs of prescription updates for accountability.

Only authorized medical staff can perform changes.

 Patient
User Story 3

As a patient, I want to schedule appointments online so that I can meet my doctor without needing to visit physically.
Priority: High
Story Points: 5

Acceptance Criteria:

Patient can view doctor availability and open time slots.

Patient can book, modify, or cancel appointments.

Confirmation is sent via email/SMS/app notification.

Appointment details are visible in the patient dashboard.

User Story 4

As a patient, I want to view my personal medical records so that I can monitor my health over time.
Priority: Medium
Story Points: 3

Acceptance Criteria:

Patient can access medical history securely after login.

Records include prescriptions, test results, and doctor notes.

Patient can only view records — no editing allowed.

Data remains confidential and secure.

 Admin
User Story 5

As an admin, I need to control and manage user accounts so that the system stays secure and well-maintained.
Priority: High
Story Points: 5

Acceptance Criteria:

Admin can create, update, or deactivate patient and doctor accounts.

Permissions and access levels can be modified.

All admin actions are logged for security tracking.

Flagged or inactive accounts can be disabled.

User Story 6

As an admin, I want to generate system-usage reports so that I can track platform performance and user activity.
Priority: Medium
Story Points: 4

Acceptance Criteria:

Admin can choose a reporting period (daily/weekly/monthly).

Reports include user activity, appointment counts, and login statistics.

Reports can be downloaded in PDF/Excel formats.

Only authorized admin users can generate reports.
